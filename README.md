# GCSE App

Student-facing GCSE revision app. **MVP: AQA GCSE Mathematics (8300).**

## Features

- **Student profile** — exam board, tier, target grade, exam date, weekly study time
- **Specification map** — 6 units, 20 topics, 50+ learning objectives
- **Daily plan** — 10 + 20 + 20 + 5 minute study loop
- **Practice** — published questions with 3-level AI hints (mark-scheme aware)
- **Mistake book** — spaced review queue with cause classification
- **Past paper mode** — mini mock with grade estimate (MVP placeholder UI)
- **Content QA** — `npm run check:content` for syllabus/question gaps

## Quick start

### 1. Install

```bash
cd gcse-app
npm install
```

### 2. Supabase

1. Create a project at [supabase.com](https://supabase.com)
2. Run `supabase/migrations/001_gcse_student_schema.sql` in the SQL Editor
3. Optionally run `supabase/seed.sql` for sample syllabus nodes and one question

### 3. Environment

```bash
cp .env.local.example .env.local
```

Fill in Supabase URL, anon key, and optionally `OPENAI_API_KEY` for live AI hints.

### 4. Run

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) → sign up → complete onboarding.

## Project structure

```
content/
  syllabus/aqa-maths-8300.json   # Source of truth for topic tree
  questions/*.json               # Question bank (import to DB)
  reports/                       # Generated by check:content
scripts/
  check-content-gaps.ts          # Daily content QA
supabase/
  migrations/001_*.sql           # Student schema + RLS
  seed.sql
src/
  app/(student)/                 # Dashboard, map, practice, mistakes
  app/api/                       # tutor hints, daily plan, feedback webhook
  lib/                           # syllabus, plan engine, grading, AI prompts
```

## Content workflow

1. Add questions to `content/questions/<objective-id>.json`
2. Run `npm run check:content` — fails if objectives lack 3 questions, hints, or mark schemes
3. Import to Supabase with `status = published` only after human review
4. Cursor Automation runs `check:content` daily and opens a fix PR

## Cursor Automations

See `.cursor/automations/` for draft configs. Recommended automations:

| Automation | Trigger | Action |
|------------|---------|--------|
| Daily content check | Weekdays 6:00 | Run gap script, open PR |
| PR review | PR opened | Smoke test auth, API, build |
| Weekly syllabus | Monday 9:00 | Diff syllabus JSON |
| Feedback webhook | HTTP webhook | Create fix task from student feedback |
| Nightly QA | Daily 23:00 | lint, build, content check |
